# Track 規格說明書: Web 演講模式與手機遙控 (Presenter Mode & Remote)

## 1. 概述 (Overview)
本 Track 旨在將 MD2PPT-Evolution 從一個靜態內容生成器轉化為具備專業展示能力的工具。透過實作雙螢幕演講者主控台與 P2P 手機遙控功能，讓使用者能在不依賴第三方軟體的情況下，直接透過瀏覽器完成專業的簡報演講。

## 2. 功能需求 (Functional Requirements)

### 2.1 雙螢幕演講模式 (Dual-Screen Presentation)
- **啟動機制**：使用者點擊「開始演講」時，系統透過 `window.open()` 開啟一個新的「觀眾視圖 (Audience View)」分頁。
- **視窗同步**：利用 `BroadcastChannel` API 實現主控台與觀眾視圖之間的即時通訊（翻頁、黑屏等）。
- **演講者主控台 (Presenter Console)**：
    - **當前與下一頁預覽**：顯示當前投影片與下一張投影片的縮圖。
    - **備忘錄系統**：解析並顯示 Markdown 中的演講者備忘錄 (`<!-- note: ... -->`)。
    - **計時器與進度**：包含總經過時間、當前時間以及分頁進度 (e.g., 5 / 20)。
    - **觀眾視線確認**：提供一個縮小的窗格顯示觀眾視圖的即時內容。

### 2.2 P2P 手機遙控器 (WebRTC Remote)
- **連線機制**：
    - 主控台生成包含 Peer ID 的 QR Code。
    - 使用者用手機掃描後進入手機版遙控頁面。
    - 透過 PeerJS (公共 Cloud Server) 建立 P2P WebRTC 連線。
- **手機操作功能**：
    - **基本導覽**：支援上一頁、下一頁與首頁跳轉。
    - **移動備忘錄**：在手機螢幕上即時閱讀當前分頁的備忘錄。
    - **黑屏模式**：一鍵控制觀眾螢幕切換為全黑。

## 3. 非功能需求 (Non-Functional Requirements)
- **無伺服器架構**：堅持 100% 純前端，不使用自架後端伺服器進行數據傳輸。
- **隱私安全性**：P2P 連線確保演講指令僅在演講者裝置之間傳輸。
- **回應式設計**：手機遙控介面需針對各種手機螢幕尺寸進行優化，按鈕應具備足夠的觸控面積。

## 4. 驗收標準 (Acceptance Criteria)
- [ ] 成功從編輯器開啟「觀眾視圖」並自動進入全螢幕模式。
- [ ] 主控台切換頁面時，觀眾視圖應在 100ms 內同步更新。
- [ ] 手機掃描 QR Code 後能成功與電腦建立 Peer 連線。
- [ ] 手機端按下下一頁時，電腦端的主控台與觀眾視圖皆同步跳轉。
- [ ] 手機端能正確讀取並顯示對應頁面的備忘錄內容。

## 5. 超出範圍 (Out of Scope)
- 虛擬雷射筆與手勢操作（保留至下一階段開發）。
- 錄製演講音訊或視訊的功能。
